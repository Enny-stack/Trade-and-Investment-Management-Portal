# Workflow — TIMP

## A) Event lifecycle
Draft → Published → Live → Post-Event → Archived

- Draft: Admin configures metadata and uploads programme/materials.
- Published: Visible on public calendar; registration opens.
- Live: Attendance + engagement tracking active.
- Post-Event: Publish recordings/decks; surveys; Deal Room continues.
- Archived: Institutional memory retained; searchable for audit (Phase 2 UI).

## B) Registration lifecycle
Invited (optional) → Registered → Verified → Active → Suspended (optional)

- Registered: user submits registration form.
- Verified: admin approves (or automated verification rule).
- Active: can access permitted materials/Deal Room.
- Suspended: admin revokes access for policy reasons.

## C) Opportunity lifecycle (Deal Room)
Draft → Published → Engaged → In Follow-up → Closed (Won/Lost)

- Draft: internal preparation.
- Published: visible in Deal Room to verified users.
- Engaged: prospectus requested and/or meetings scheduled.
- In Follow-up: internal follow-up actions created and tracked.
- Closed: outcome captured (won/lost/paused).

## D) Engagement events (what we track)
- Event page view
- Registration submitted / verified
- Material published (admin) + material viewed/downloaded (user)
- Deal page view
- Prospectus request
- Meeting scheduled
- Survey completed

## E) Key admin actions
- Publish/unpublish events
- Approve/reject registrations
- Publish materials and trigger notifications
- Publish/unpublish deals
- Export analytics
